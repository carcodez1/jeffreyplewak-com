# Security Policy

This repository powers a public personal site.

Security and stability are taken seriously.

---

## Reporting Issues

If you discover a security issue, please:

- Open a GitHub issue labeled "security", or
- Contact via LinkedIn if the issue involves sensitive exposure

Please provide:

- A clear description
- Reproduction steps
- Expected vs actual behavior

---

## Scope

In scope:

- XSS vulnerabilities
- CSP bypasses
- Metadata leaks
- Deployment misconfiguration
- CI workflow abuse

Out of scope:

- Social engineering attempts
- Content disagreements
- Performance preference debates

---

## Dependency Management

- Node version is pinned to 20.x
- CI actions are versioned
- Runtime dependencies are kept minimal

---

## Disclosure

Security issues will be addressed pragmatically and patched directly to `main` when necessary.
