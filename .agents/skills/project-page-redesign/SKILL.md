# Project Page Redesign

Author: Jeffrey R. Plewak
Created: 2026-03-15
License: Internal repo guidance

When to use:
- an existing `/projects/*` route needs layout, hierarchy, or visual cleanup
- the goal is stronger scanability without changing the underlying claims

Workflow:
1. Inspect the current proof page, related cross-links, and the route's strongest evidence.
2. Preserve section purpose and claim meaning; simplify structure before adding visual treatment.
3. Keep the page calm, directional, and proof-first rather than recruiter-funnel-like.
4. Avoid theme drift and unnecessary client-only rendering.

Verify:
- `npm run -s verify:patch`

Deliverables:
- scoped redesign diff
- exact verification results
